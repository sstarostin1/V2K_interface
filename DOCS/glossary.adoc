== Основные термины

=== VCAS (VEPP Channel Access Server)
Сервер каналов комплекса синхротрона ВЭПП-2000. Предоставляет доступ к данным различных подсистем ускорителя в реальном времени.

=== Канал (Channel)
Логическая единица данных в системе VCAS. Представляет параметр ускорителя (энергия, ток, температура и т.д.). Каналы имеют иерархическую структуру имен и типы данных.

=== Клиент (Client)
Приложение, подключающееся к VCAS серверу для получения данных. VCAS Server Viewer является клиентским приложением.

=== Мок-сервер (Mock Server)
Тестовый сервер-заглушка, имитирующий работу реального VCAS сервера. Используется для разработки и тестирования без подключения к реальному комплексу.

=== Временные ряды (Time Series)
Последовательность значений канала с временными метками. Используются для построения графиков изменения параметров во времени.

=== Drag-and-Drop
Метод взаимодействия с интерфейсом, при котором пользователь может перетаскивать каналы из дерева на графики для добавления.

== Типы каналов VCAS

=== RO (Read-Only)
Канал только для чтения. Значение устанавливается аппаратной частью или другими системами.

=== RW (Read-Write)
Канал для чтения и записи. Позволяет изменять значение параметра через клиентское приложение.

=== EX (Exclusive)
Эксклюзивный канал. Доступен только для монопольного использования одной системой или пользователем.

== Компоненты интерфейса

=== Дерево каналов (Channel Tree)
Иерархический список каналов, организованный по подсистемам ускорителя (VEPP, BEP, Magnets и т.д.).

=== Окно информации (Info Panel)
Панель, отображающая детальную информацию о выбранном канале (тип, единицы измерения, текущее значение).

=== Окно графиков (Plot Window)
Независимое окно с графиками временных рядов каналов. Поддерживает множественные каналы в одном окне.

=== Контейнер графиков (Plot Container)
Компонент управления компоновкой и разделением областей графиков внутри окна.

=== Dock Window
Отдельное, независимое окно, которое можно размещать внутри главного окна или отдельно.

== Технические термины

=== PyQt5
Кроссплатформенный фреймворк для создания графических интерфейсов на Python. Используется для реализации всего пользовательского интерфейса приложения.

=== QTcpSocket
Класс Qt для работы с TCP соединениями. Используется в VCASClient для подключения к серверу.

=== Сигнално-слотовая модель (Signals & Slots)
Механизм асинхронного взаимодействия объектов в Qt. Используется для обновления интерфейса при получении новых данных.

=== Numpy
Библиотека для работы с многомерными массивами. Используется для эффективной обработки данных каналов для графиков.

=== Logging
Система сбора и записи сообщений о работе программы. Поддерживает различные уровни логирования (DEBUG, INFO, WARNING, ERROR, CRITICAL).

== Протокол VCAS

=== Текстовый протокол
Простой протокол обмена сообщениями в формате "ключ:значение|ключ:значение". Используется для всех команд и ответов.

=== Subscribe
Команда подписки на обновления канала в реальном времени. После подписки сервер автоматически отправляет новые значения.

=== GetHistory
Команда получения исторических данных канала за указанный период времени.

=== GetFull
Команда получения полной информации о канале (тип, единицы, описание, текущее значение).

== Настройки и конфигурация

=== Config
Класс управления настройками приложения. Сохраняет параметры между запусками в JSON файл.

=== Window Config
Файлы конфигурации окон графиков (.config.json). Содержат список каналов, настройки времени и другие параметры графика.

=== Data Exports
Экспортированные данные каналов. Могут сохраняться в CSV или JSON форматах для дальнейшего анализа.

== Режимы работы

=== Рабочий режим (Production Mode)
Работа с реальным VCAS сервером комплекса ВЭПП-2000. Требует доступности сервера в сети.

=== Режим отладки (Debug Mode / Mock Mode)
Работа с тестовым сервером-заглушкой. Используется для разработки, тестирования и демонстрации.

=== Автономный режим (Offline Mode)
Работа с сохраненными данными. Возможен анализ экспортированных ранее данных.

== Архитектурные паттерны

=== MVC-подобная архитектура
Разделение ответственности на Модель (ChannelData), Вид (MainWindow, Plot widgets) и Контроллер (VCASClient, PlotManager).

=== Синглтон
Шаблон, при котором класс имеет только один экземпляр. Используется в Config для глобальных настроек.

=== Фабричный метод
Шаблон для создания объектов. Может применяться при создании различных типов каналов или виджетов графиков.

=== Наблюдатель (Observer)
Шаблон, реализуемый через сигналы/слоты Qt. Используется для реактивного обновления интерфейса.

== Сокращения

=== ВЭПП-2000
Высоковольтный электронно-позитронный коллайдер второго поколения - ускорительный комплекс в Новосибирске.

=== BPM
Beam Position Monitor - монитор положения пучка.

=== FCT
Fast Current Transformer - быстрый трансформатор тока.

=== PMT
Photo Multiplier Tube - фотоэлектронный умножитель (ФЭУ).

=== GUI
Graphical User Interface - графический пользовательский интерфейс.

=== API
Application Programming Interface - программный интерфейс приложения.

=== CSV
Comma-Separated Values - формат файлов с разделителями-запятыми.

=== JSON
JavaScript Object Notation - формат сериализации данных.

=== TCP
Transmission Control Protocol - протокол управления передачей в компьютерных сетях.
