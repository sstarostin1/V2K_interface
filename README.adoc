== Назначение проекта

VCAS Server Viewer - специализированное графическое приложение для мониторинга и анализа архитектуры сервера каналов VCAS в ускорительном комплексе ВЭПП-2000. Приложение предоставляет исследователям и инженерам удобный интерфейс для:

* Просмотра иерархической структуры каналов сервера
* Мониторинга значений каналов в реальном времени
* Визуализации временных рядов данных
* Отладки и тестирования системы без подключения к реальному серверу

== Архитектурный контекст

Приложение является частью экосистемы управления ускорительным комплексом ВЭПП-2000, где VCAS (VEPP Channel Access Server) выступает в роли сервера данных каналов. VCAS Server Viewer служит клиентским приложением для:

* Диагностики состояния каналов
* Анализа поведения системы
* Разработки и тестирования новых компонентов

== Ключевые возможности

=== Просмотр структуры каналов
* Древовидное представление иерархии каналов
* Поиск и фильтрация каналов
* Детальная информация о каждом канале (тип, значение, статус)

=== Визуализация данных
* Интерактивные графики временных рядов
* Drag-and-drop каналов между графиками
* Мульти-окно система с независимыми окнами графиков
* Настраиваемое окно времени для анализа

=== Режимы работы
* *Рабочий режим*: Подключение к реальному VCAS серверу
* *Режим отладки*: Работа с тестовым сервером-заглушкой
* *Автономный режим*: Анализ сохраненных данных

=== Интеграция с системой
* Протокол VCAS для обмена данными
* TCP-соединение с сервером
* Асинхронные обновления интерфейса
* Кэширование данных для производительности

== Технологическая основа

=== Язык и фреймворки
* **Python 3.7+** - основной язык разработки
* **PyQt5** - фреймворк для графического интерфейса
* **Qt Charts** - компоненты для построения графиков

=== Коммуникации
* **QTcpSocket** - TCP-клиент для подключения к серверу
* **Кастомный протокол VCAS** - текстовый протокол key:value|key:value

=== Архитектурные паттерны
* **MVC-подобная архитектура** - разделение данных, логики и представления
* **Сигнально-слотовая модель** - асинхронное взаимодействие компонентов
* **Модульная структура** - независимые компоненты с четкими интерфейсами

== Цели и ценности

=== Научная ценность
* Обеспечение точного мониторинга параметров ускорителя
* Поддержка экспериментальной деятельности
* Упрощение анализа данных для исследователей

=== Техническая надежность
* Стабильная работа в условиях реального времени
* Минимальные задержки при обновлении данных
* Обработка ошибок и восстановление соединения

=== Пользовательский опыт
* Интуитивный интерфейс для инженеров и физиков
* Гибкая настройка под различные сценарии использования
* Поддержка как новичков, так и экспертов

== Ограничения и допущения

* Приложение работает только в среде Windows (ограничение PyQt5)
* Требует наличия VCAS сервера для полного функционала
* Оптимизировано для работы с сотнями каналов, не тестировано на тысячах
